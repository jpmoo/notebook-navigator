# Contributing

Notebook Navigator is maintained by a single author. The project does not accept pull requests. Ideas are welcome as feature requests through GitHub issues.

## Bug Reports

Open a bug report using the [bug report template](https://github.com/johansan/notebook-navigator/issues/new?template=bug_report.md). Include the Notebook Navigator version, Obsidian version, platform, reproduction steps, and any relevant sample vault data.

## Feature Requests

Open a feature request using the [feature request template](https://github.com/johansan/notebook-navigator/issues/new?template=feature_request.md). Describe the use case and the behavior you would like to see.

## Questions And Discussion

General questions and discussion happen on [Discord](https://discord.gg/6eeSUvzEJr).

## Security Issues

Security issues follow a separate process. See [SECURITY.md](SECURITY.md).

## Pull Requests

Pull requests are not accepted. Any pull request is closed automatically without review.

With the emergence of agentic coding, code submitted from outside cannot be quality-controlled to the standard the project maintains, so all code stays under the maintainer's single design and release process. Contribute ideas as feature requests instead — bug reports and feature requests through GitHub issues are the supported way to influence the project.
